# PaperLab Professional Beta

## About This Beta

**This is a closed, private beta to validate our architecture and marking workflow as a proof-of-concept.**

PaperLab uses AI to automatically mark GCSE Mathematics past papers (Pearson Edexcel Higher tier only). This beta tests whether our technical approach works end-to-end, not whether we've achieved optimal marking accuracy.

**Important:**
- **Not for students:** This is a closed beta for testing purposes only, not for students preparing for actual exams
- **Not production-ready:** Accuracy optimization happens after we validate the core system
- **Private testing:** This beta validates our technical foundation before any public release
- **No data migration guarantee:** Beta data (accounts, uploads, marking results) may not be migrated to production. Do not rely on beta data being preserved.

## What We're Testing

This beta validates our **architecture and workflow**, specifically:

1. **Technical workflow:** Does the end-to-end process work reliably?
2. **User experience:** Is the marking process intuitive and smooth?
3. **System stability:** Does everything run without crashes or failures?
4. **Integration:** Do all components work together correctly?

We expect fair marking accuracy but are not optimizing for this yet. Accuracy refinement comes after we validate the core system.

## How to Test

### Complete Marking Workflow

Test the full paper marking process from start to finish:

#### Step 1: Select a Paper
1. From the home screen, tap "Select Paper"
2. Choose a paper type from the dropdown (e.g., "Paper 3 Higher")
3. Select an exam session (e.g., "November 2023")
4. Tap "Confirm"

This creates a paper attempt and opens the upload screen.

#### Step 2: Upload Student Work for Each Question

The upload screen shows all questions in the paper (e.g., Q1-Q20). For each question:

1. Tap a question row (e.g., "Q1")
2. Tap "Add Photo" or the camera icon
3. Choose "Take Photo" or "Choose from Library"
4. Position your camera over the student's written work for that question
5. Ensure the writing is clear and well-lit
6. Take or select the photo(s)
7. The photo automatically uploads and you return to the paper upload screen

**Tip:** Good lighting and legible handwriting help the marking process.

**Repeat for all questions.** The upload screen shows your progress (e.g., "15/20 uploaded").

#### Step 3: Submit the Complete Paper
1. Once all questions have photos uploaded, the "Confirm" button becomes enabled
2. Tap "Confirm" to submit the entire paper for marking
3. You'll see a progress screen while PaperLab marks all questions (typically 30-60 seconds for a full paper)

#### Step 4: Review Paper Results
1. View the overall grade awarded (e.g., "Grade 7")
2. See the total marks (e.g., "65/80")
3. Tap any question row to view detailed feedback for that question
4. Review the working analysis and marks awarded for individual questions
5. Note any issues or observations

## What to Report

### Technical Issues
- App crashes or freezes
- Upload failures
- Marking errors or timeouts
- Missing or broken features

### Workflow Feedback
- Is the process intuitive?
- Are instructions clear?
- Does the camera interface work well?
- Is the results screen easy to understand?

### Performance Observations
- How long does full paper marking take?
- Is photo upload smooth for each question?
- Does the app stay responsive when uploading multiple photos?
- Are there delays or lag during navigation?

### Workflow Observations
- Is it clear you need to upload photos for all questions?
- Is the progress indicator (e.g., "15/20 uploaded") helpful?
- Does the question-by-question upload flow make sense?
- Is it easy to navigate between questions?

### Marking Observations
- Note significant marking discrepancies if you see them, but remember: we're validating the system, not optimizing accuracy yet
- Check if grades and total marks make sense
- Review individual question feedback quality
